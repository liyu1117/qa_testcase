# UI 测试用例规范

## 用例编写标准

### 用例结构
每条 UI 测试用例必须包含以下要素：

| 要素 | 必填 | 说明 |
|------|------|------|
| 用例编号 | 是 | 格式: UI-{页面缩写}-{序号}，如 UI-LOGIN-001 |
| 标题 | 是 | {页面名称} - {检查项描述} |
| 模块 | 是 | 页面路径 |
| 优先级 | 是 | P0/P1/P2/P3 |
| 检查维度 | 是 | 视觉还原 / 交互行为 / 响应式 / 无障碍 |
| 设计稿参考 | 是 | Figma 链接和 Frame 名称 |
| 检查清单 | 是 | 表格形式：检查项、检查方式、设计规范值、通过标准 |
| 交互行为验证 | 否 | 表格形式：触发条件、操作、预期响应、动效要求 |
| 响应式断点验证 | 否 | 不同分辨率下的布局预期 |

### 检查维度说明
1. **视觉还原**：颜色、字号、间距、图标与设计稿一致性
2. **交互行为**：点击、悬停、聚焦、滚动等交互响应
3. **响应式**：不同屏幕尺寸下的布局适配
4. **无障碍**：键盘导航、屏幕阅读器支持、对比度
